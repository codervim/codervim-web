         VIM 与代码高亮、自动补全
  
  Code complete assistant ios one basic function of the coding editor tools.
    代码补全，应该是程序员对编辑器最为关注的一件事情了吧。从我对身边的程序员的观察来说，多数人对
VIM，Emacs
这类编辑器的第一感觉就是不能代码提示，很多用了VIM的人又说VIM是可以做到，同时VIM又做的不够好。
那么VIM和IDE的自动补全有什么不同呢？是怎么实现的呢？ 
高亮和自动补全有什么关联呢？

我们先来说高亮。
这是一个非常有意思的事情，如果软件工程的学生毕业了，还不知道编辑器代码自动补全是怎么实现的，那多少就有点
问题了。

我们先来看看VIM，大家可以看看这个文件，/usr/share/vim/vim73/syntax/python.vim 
如果把这个文件删除了，VIM还能对Python脚本语法高亮吗？这时，其他语言的脚本还可
当然是不能了，关键就在这个文件上。看这个文件的内容，就能够初步的才想到，
语法高亮实际上是工作在正则层面上的。


虽然我没有去考察过IDE，但是猜测肯定是代码检查和语法高亮是一块儿做的。


为什么Java、C#的智能代码提示，或者说代码自动补全功能非常强大，极其准确，
Visual Studio 对C++的支持也非常不错，因为有C语法的关系，效果还是差了那么一点，
Eclipse CDT 对C++的支持慢慢的变好了，但是，使用过的人应该都有感觉：
不够准确啊。 产生这些差异的原因是什么呢？  IDE智能提示使用了什么原理呢？


我们也是先来看一看VIM 的“智能提示”是怎么做到的，大家看到了，这里有个引号。
  VIM 自身并不提供“智能提示”的功能，需要以插件的形式来提供，IDE大多是集成
这样的功能的，不需要使用者操心。
